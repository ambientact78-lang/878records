Firebase Setup Guide

Step 1 — Go to console.firebase.google.com and create a free project.

Step 2 — Enable Authentication → Email/Password sign-in.

Step 3 — Create your admin account under Authentication → Users → Add user.

Step 4 — Enable Firestore Database (start in test mode, then add rules).

Step 5 — Go to Project Settings → Your Apps → Add Web App. Copy the config object and paste it into both store.html and admin.html where it says REPLACE_….

Step 6 — In Firestore, create a collection called beats. Each beat document should have: title, artist, bpm, key, genre, price, audioUrl, gumroadUrl, order.

Recommended Firestore Rules:
rules_version = '2'; service cloud.firestore { match /databases/{db}/documents { match /beats/{id} { allow read: if true; allow write: if request.auth != null; } match /{doc=**} { allow read, write: if request.auth != null; } } }

Audio Hosting (Free Options)

Since Firebase Storage has a free 5GB limit, you can host your beat previews there, or use these free alternatives:

• Google Drive — Upload MP3, get a direct link (use a converter like gdocs2direct)
• Dropbox — Share link, change ?dl=0 to ?dl=1 for direct play
• SoundCloud — Embed player URL from SoundCloud
• GitHub — Host MP3 files in a repo, use raw GitHub URL

Paste the direct MP3 URL into the Audio URL field when adding a beat.
