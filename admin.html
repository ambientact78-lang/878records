🔴 Beats Not Showing on Store Page?

Root cause: Firestore's where + orderBy on different fields requires a composite index.

Fix (Option 1 — Already done): The updated store.html now sorts beats client-side, so no index is needed. Just upload the new store.html.

Fix (Option 2 — Create the index):
1. Open browser Console (F12) on the store page
2. Find the Firebase error — it has a direct link to create the index
3. Click that link in Firebase Console → click "Create Index"
4. Wait ~60 seconds, reload store page

Free Audio Hosting

• Firebase Storage (recommended) — Free 5GB. Console → Storage → Upload file → copy Download URL
• GitHub — Push MP3 to repo → use raw.githubusercontent.com/USER/REPO/main/file.mp3
• Google Drive — Upload → Share → use gdocs2direct.com to convert to direct link
• Dropbox — Share link → change dl=0 to raw=1
